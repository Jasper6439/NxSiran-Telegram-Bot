# v1.9 API Token 迁移总结

## 概述

本次迁移将 `api_tokens.json` 从文件存储迁移到 SQLite 数据库，彻底贯彻"单一数据源"原则。

## 主要变更

### 1. 数据库 Schema 变更

#### 新增表：`api_tokens`
```sql
CREATE TABLE api_tokens (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    token_hash TEXT NOT NULL UNIQUE,
    created_at TEXT NOT NULL,
    expired_at TEXT,
    is_revoked INTEGER DEFAULT 0,
    last_used_at TEXT,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

#### 级联删除约束（ON DELETE CASCADE）
以下表已添加级联删除约束，当用户被删除时，关联数据自动清理：

## SQLite 线程安全与事务配置

### 线程安全
```python
conn = sqlite3.connect(
    self.db_path,
    timeout=30,
    check_same_thread=False  # 允许多线程访问
)
```

配合 `threading.local()` 使用，确保每个线程有自己的数据库连接实例。

### 事务保护
关键操作（如金币扣除）使用数据库事务保证原子性：
```python
with self.get_connection() as conn:
    # 查询当前余额
    cursor = conn.execute("SELECT money FROM farms WHERE user_id = ?", (user_id,))
    row = cursor.fetchone()
    
    # 检查余额充足
    if row['money'] < amount:
        return {'success': False, 'error': '金币不足'}
    
    # 执行扣除
    conn.execute("UPDATE farms SET money = ? WHERE user_id = ?", (new_balance, user_id))
    # 同一事务中记录日志
    conn.execute("INSERT INTO game_events ...")
```

## 负面测试用例

所有测试用例位于 `tests/test_negative_cases.py`：

1. **无效 Token 访问** - 返回 401
2. **不存在用户访问** - 返回 404
3. **缺少认证头** - 返回 401
4. **格式错误认证头** - 返回 401
5. **已撤销 Token** - 返回 401
6. **过期 Token** - 返回 401
7. **金币扣除事务** - 确保原子性
8. **级联删除验证** - 确保外键约束正确

## 迁移步骤

1. **执行 SQL 迁移**
   ```bash
   sqlite3 database/data/game.db < database/data/migration_v19_api_tokens.sql
   ```

2. **导入现有数据**
   ```bash
   python3 tools/migrate_api_tokens_to_db.py
   ```

3. **验证迁移**
   ```bash
   python3 -m pytest tests/test_negative_cases.py -v
   ```

4. **清理旧文件**（可选）
   ```bash
   rm data/api_tokens.json
   ```

## 向后兼容性

- `api/deps.py` 仍保留对 `api_tokens.json` 的读取支持（后备机制）
- 新创建的 Token 只存储在数据库中
- 旧的 JSON Token 迁移后不再使用

## 安全注意事项

1. **Token 存储**：只存储 SHA256 hash，不存储原始 Token
2. **过期机制**：支持设置 Token 过期时间
3. **撤销机制**：支持主动撤销 Token
4. **审计日志**：记录 Token 创建、使用和撤销操作
