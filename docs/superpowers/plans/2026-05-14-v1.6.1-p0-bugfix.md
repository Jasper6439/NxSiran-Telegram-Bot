## Task 2: 修复 SQL Schema INSERT 语句缺陷

**问题:** `game_schema.sql` 第 265 行的 `INSERT OR IGNORE INTO crop_types` 语句在第 273 行以分号结束，但第 276-294 行的后续 VALUES 行缺少 `INSERT OR IGNORE INTO crop_types` 前缀，导致 SQL 语法错误。

**根因:** 第 265 行的 INSERT 语句只包含基础作物（第 266-273 行），以分号结尾。第 276-294 行的 v1.4.10 新增作物 VALUES 没有对应的 INSERT 前缀。

### Step 1: 修复 game_schema.sql

**文件:** `game_schema.sql`
**行号:** 276

在第 276 行之前添加新的 INSERT 语句前缀，并修复 VALUES 列数（新增作物多了一个 `tier` 列）。

```diff
  -- v1.4.10 新增作物 - 进阶系统
  -- 快速作物 (1-30分钟) - 新手友好
+ INSERT OR IGNORE INTO crop_types (id, name, name_ko, growth_time, sell_price, seed_price, seasons, emoji, description, tier) VALUES
  ('sunflower', '向日葵', '해바라기', 5, 15, 5, '["spring", "summer"]', '🌻', '阳光的向日葵', 1),
```

**注意:** 需要确认 `crop_types` 表定义中是否包含 `tier` 列。如果表定义中没有 `tier` 列，则需要：
1. 先在 `CREATE TABLE crop_types` 中添加 `tier INTEGER DEFAULT 0` 列定义
2. 或者将 VALUES 中的 tier 值移除

### Step 2: 验证

```bash
# 使用 sqlite3 验证 schema 语法
sqlite3 :memory: < game_schema.sql && echo "Schema OK" || echo "Schema ERROR"
```

### 提交信息

```
fix: 修复 game_schema.sql 中 v1.4.10 新增作物 INSERT 语句缺少前缀

第 276-294 行的 VALUES 缺少 INSERT OR IGNORE INTO crop_types 前缀，
导致 SQL 执行失败。
```

## Task 3: 修复 Schema 路径不匹配

**问题:** `database/base.py` 第 22 行 `SCHEMA_PATH` 指向 `database/game_schema.sql`（与 base.py 同目录），但实际文件位于项目根目录 `game_schema.sql`。`database/` 目录下没有 `data/` 子目录。

**当前代码:**
```python
# database/base.py 第 22 行
SCHEMA_PATH = os.path.join(os.path.dirname(__file__), 'game_schema.sql')
```

`os.path.dirname(__file__)` 解析为 `database/`，所以 SCHEMA_PATH = `database/game_schema.sql`，但文件实际在 `game_schema.sql`（项目根目录）。

### Step 1: 移动 game_schema.sql 到正确位置

```bash
mkdir -p database/data
mv game_schema.sql database/data/game_schema.sql
```

### Step 2: 更新 database/base.py 的 SCHEMA_PATH

**文件:** `database/base.py`
**行号:** 22

```diff
- SCHEMA_PATH = os.path.join(os.path.dirname(__file__), 'game_schema.sql')
+ SCHEMA_PATH = os.path.join(os.path.dirname(__file__), 'data', 'game_schema.sql')
```

### Step 3: 验证

```bash
# 验证文件存在
ls -la database/data/game_schema.sql

# 验证路径解析
python3 -c "
import os
schema = os.path.join(os.path.dirname('database/base.py'), 'data', 'game_schema.sql')
print(f'SCHEMA_PATH: {schema}')
print(f'Exists: {os.path.exists(schema)}')
"

# 验证数据库初始化
python3 -c "from database.base import SCHEMA_PATH; print(SCHEMA_PATH); import os; print('OK' if os.path.exists(SCHEMA_PATH) else 'MISSING')"
```

### 提交信息

```
fix: 修复 database/base.py SCHEMA_PATH 路径不匹配

- 将 game_schema.sql 移动到 database/data/ 目录
- 更新 SCHEMA_PATH 指向 database/data/game_schema.sql
```
